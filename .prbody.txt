## 摘要
实现 OpenSpec change `m0-scaffold-tauri-skeleton`（技术方案 M0 项目骨架）。Tauri v2 + Vite + React + TS 桌面外壳：三窗口、托盘、应用数据目录、跨窗口事件总线、文件日志。无业务逻辑（事件服务 / 宠物 / 通知 / Adapter 属 M1–M6）。

## 主要改动
- 前端：`pet-overlay`（透明 Canvas 占位 + `startDragging` 拖动）、`status-panel`、`settings` 三个独立 webview 入口（Vite 多页）；`health.ts` 订阅 `agentpet://health`。
- 后端：`lib.rs`（窗口/托盘/`CloseRequested` 关闭到托盘/退出/生命周期日志）、`app_state.rs`、`commands.rs`（health）、`config/{paths,settings}`、`window/overlay`、`logging`；`event_bus/session/adapters/pet/notify/store` 占位模块。
- 配置：`tauri.conf.json` 三窗口 + overlay 透明/置顶/无边框/skip-taskbar；`capabilities/default.json` 限定三窗口 + `core:window:allow-start-dragging` + `core:event:default`；移除未用的 `plugin-opener`。

## 验证
- `npm run build`：tsc 类型检查 + vite 多页构建通过。
- `cargo test`：全量编译（含 `generate_context!` 校验 conf/capabilities）+ paths/settings 单元测试通过（2 passed）。
- **待人工 GUI 确认**（agent 无法观察 GUI）：1.4 启动窗口、3.4 运行时日志、4.6 窗口属性/拖动、5.5 关闭到托盘/退出、6.5 health 双窗口广播 —— 运行 `npm run tauri dev` 验证。
